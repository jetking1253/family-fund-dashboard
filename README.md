# 家庭基金管理系统综合看板 (Family Fund Management Dashboard)

这是一个基于 `Python` + `Streamlit` + `Supabase (PostgreSQL)` 打造的个人及家庭级资产配置与净值管理工具。随着版本的迭代（V1 -> V5），系统不仅包含了严谨的会计净值核算，还逐步引入了公网鉴权保护、大语言模型人工智能 OCR 视觉解析、原子级数据容错倒流机制，以及彻底的云端化部署能力。

## 🌟 核心理念
通过统一的“系统总净资产”和“系统总份额”，精确计算每一次成员资金进出（定投或赎回）对个人财富池占比的影响。采用专业的基金级「单位净值算法」，彻底解决由于各个成员投入时间不同、投入金额不同而带来的绝对收益分配不均带来的财务记账难题。

## 🚀 系统功能演进全览

### 🔹 V1 - 坚实的核算地基
- **数据库隔离设计**: 数据层 (`db.py`)、业务逻辑层 (`fund_logic.py`)、视图交互层 (`app.py`) MVC 级解耦。
- **高阶净值算法**: 任何新的资金注入必先通过**重估资产总市值**得出实时更新的单价净值 (NAV)，之后再基于新 NAV 生成成员应得的新增份额。系统数字精度统一保持至小数点后四位。
- **丰富的可视化展现**: 利用 `Plotly` 提供直观清晰的系统走势折线图，及仓位资产分布饼图。

### 🔹 V2 - 智能录入与安全回滚
- **"时光倒流"事务容错**: 开辟 **📜 历史与回滚** 专区。提供能将误操作导致净值暴跌/飙升的**错账一键撤销**功能。系统会在底层同步追溯销毁由该操作衍生的所有流水和多余的人造份额。
- **AI 视觉自动录表 (OCR)**: 告别繁琐的手工填数字，接入大语言模型强大的多模态图片阅读能力，只需上传券商/银行资金转移截图，底层自动抓取金额并回填至表单。防呆机制确保只有人工复核后才最终入库。

### 🔹 V3 - 泛模型支持与人事管理拓展
- **自定义大模型泛化支持**: 脱离单一的 Google 平台依赖，侧边栏支持任意兼容 **OpenAI 标准**的 `Chat Completions Vision` 大模型引擎（含各类国产开源及闭源的优质多模态模型 API）。
- **家庭树拓扑管理**: 解锁 **👥 家庭成员管理中心**。在不丢失原有人员的真实流水财务关联之下，支持为系统新增空白占位成员，以及灵活修改现任操作人的昵称别名。

### 🔹 V4 - 公网强加固壁垒
- **安全拦截鉴权体系**: 因顺应公网云端部署计划，系统最外部架设了 `Session` 和基于密码散列加密技术的登录视图阻截墙。
- **独立的账号密码本**: 数据后台新切独立的管理员和子账本数据库表 (`app_users`) 容解。超级管理员不但可以更新自己的密码锁存器并实施热替，亦可随时给下放查验与做账需求的其他家庭账本成员提供不涉权的自定义登录身份。

### 🔹 V5 - 全面云端化与 Supabase 改造
- **数据库架构升级**: 彻底废弃本地 SQLite 数据库，将底层架构全量迁移至高性能云端数据库 `Supabase (PostgreSQL)`。
- **Streamlit Cloud 无缝部署**: 适配 Streamlit Community Cloud 原生 Secrets 凭证管理，实现本地与云端生产环境的配置解耦。
- **数据访问层重构**: 使用 `supabase-py` 客户端的链式调用语法重构了 `db.py`，保持了原有的所有核心功能和事务级流转在云化后的完美兼容。

## 💾 运行部署指令

1. 环境依赖拉取：
   ```bash
   pip install -r requirements.txt
   ```

2. 数据库准备：
   - 在 [Supabase](https://supabase.com/) 创建项目。
   - 复制项目根目录的 `supabase_schema.sql` 并在 Supabase 的 SQL Editor 中运行以创建所有表和初始数据。

3. 配置凭证：
   - 修改项目根目录的 `.streamlit/secrets.toml`，在文件中填入您的真实 Supabase URL 与 API Key。*(如果不存在该文件夹，请自行创建)*。
   *(如部署到 Streamlit Community Cloud，请在 App 设置的高级选项 Secrets 中填入上述 TOML 格式的配置即可。)*

4. 在项目根目录，通过如下指令启动：
   ```bash
   streamlit run app.py
   ```
*(系统初始化时会预植入初始超级账户 `admin` - 默认访问口令为 `admin123`，以此保证顺利穿透公网鉴权)*
